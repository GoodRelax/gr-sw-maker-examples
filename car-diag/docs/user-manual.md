## 1. はじめに

car-diag は、ELM327 Bluetooth ドングルを使って車両の診断情報を取得・表示・記録できる Windows 11 用デスクトップアプリケーションです。

### 主な機能

- **故障コード（DTC）の読取と消去** — エンジン警告灯が点灯したときの原因コードを確認し、消去できます。
- **リアルタイムデータの表示** — エンジン回転数・水温・車速などの車両データをリアルタイムで確認できます。
- **データの記録** — 走行中のデータをファイルに保存し、あとから Excel などで分析できます。
- **ECU の自動検出** — 車両に搭載されている電子制御ユニット（ECU）を自動的に検出します。
- **日本語インターフェース** — すべての操作画面と表示が日本語です。

### 対応環境

| 項目 | 要件 |
|------|------|
| OS | Windows 11 |
| 接続方式 | Bluetooth（ELM327 ドングル経由） |
| 対応プロトコル | Legacy OBD-II、UDS（ISO 14229）、KWP2000（ISO 14230） |
| Python（pip 版のみ） | 3.12 以上 |

### 対応していない機能

以下の機能には対応していません。

- USB 接続の ELM327
- フラッシュ書換・アクチュエータ制御などのメーカー固有拡張サービス
- クラウド連携・リモート監視
- macOS / Linux

## 2. 準備

### 2.1 ELM327 ドングルの購入ガイド

ELM327 ドングルは、車の OBD-II ポートに差し込んで車両と通信するアダプターです。Bluetooth 接続タイプを購入してください。

#### 推奨チップ

**ELM Electronics 社製の本物のチップ**（ELM327 v1.5 以降）を搭載した製品を強く推奨します。

OBD-II アダプターには、本物の ELM327 チップではなく、互換品（クローン品）のチップを搭載した安価な製品が多く流通しています。

#### クローン品の注意点

クローン品は外見が本物と区別できませんが、以下のような問題が起きる場合があります。

- 接続が確立できない（「UNABLE TO CONNECT」エラーが発生する）
- 特定のコマンドに正しく応答しない
- データが途切れる、または誤ったデータが返ってくる

クローン品で動作しない場合は、信頼できるメーカーの製品への買い替えを検討してください。car-diag はクローン品の動作を保証しません。

#### 購入時の確認ポイント

- Bluetooth 接続タイプであること（Wi-Fi タイプは対応外）
- パッケージや商品説明に「ELM327」と記載されていること
- OBD-II（16 ピン）コネクター形状であること

---

### 2.2 OBD-II ポートへの接続

#### OBD-II ポートの場所

OBD-II ポートは、ほとんどの乗用車でダッシュボード下部（運転席側）に設置されています。ハンドル柱の下あたりを探してください。形状は台形の 16 ピンコネクターです。

日本国内で販売された 2001 年以降の乗用車には、OBD-II ポートが搭載されています（一部の軽自動車を除く）。

#### 挿し込み方

1. エンジンを切った状態でドングルをポートに差し込みます。
2. カチッと固定されるまでしっかり押し込みます。
3. ドングルの LED が点灯すれば電源が入っています（製品により異なります）。

エンジンをかけなくても、イグニッションを ACC または ON の位置にするだけでドングルに電源が供給されます。

---

### 2.3 Windows 11 での Bluetooth ペアリング

car-diag はアプリ内でのペアリング操作に対応していません。必ず先に Windows の設定からペアリングを行ってください。

#### ペアリング手順

1. ドングルを OBD-II ポートに挿し、イグニッションを ACC または ON にします。
2. Windows 11 の「スタートメニュー」を開き、「設定」をクリックします。
3. 「Bluetooth とデバイス」をクリックします。
4. 「Bluetooth」がオンになっていることを確認します。
5. 「デバイスの追加」をクリックします。
6. 「Bluetooth」を選択します。
7. デバイス一覧に「OBDII」「ELM327」などの名前が表示されたらクリックします（製品により名称が異なります）。
8. PIN の入力を求められたら **1234** を入力して「接続」をクリックします（製品によっては「0000」の場合もあります）。
9. 「デバイスの準備が整いました」と表示されればペアリング完了です。

ペアリングが完了すると、「Bluetooth とデバイス」の画面にデバイスが表示されます。

---

### 2.4 COM ポートの確認

Bluetooth ペアリングが完了すると、Windows がドングルを仮想 COM ポートとして認識します。car-diag で接続するには、その COM ポート番号を確認しておく必要があります。

#### デバイスマネージャーでの確認手順

1. キーボードの **Windows キー + X** を押し、「デバイスマネージャー」をクリックします。
2. 「ポート（COM と LPT）」をクリックして展開します。
3. 「標準シリアル over Bluetooth リンク（COM○）」または「Bluetooth シリアル（COM○）」などの名前が表示されます。この **COM○** の番号が使用する COM ポートです。

複数の COM ポートが表示される場合があります。ELM327 ドングルに対応するポートは「送信（Outgoing）」側の COM ポートです。

## 3. インストール

car-diag のインストール方法は、**exe ファイル版**と **pip インストール版**の 2 種類があります。Python に不慣れな方は exe 版を推奨します。

### 3.1 Python 環境のインストール（pip 版のみ）

exe 版を使用する場合、この手順は不要です。

1. ブラウザで [https://www.python.org/downloads/](https://www.python.org/downloads/) を開きます。
2. 「Download Python 3.12.x」（または最新の 3.12 以上のバージョン）をクリックしてインストーラーをダウンロードします。
3. ダウンロードしたインストーラーを実行します。
4. インストール画面の一番下にある「**Add Python to PATH**」にチェックを入れてから「Install Now」をクリックします。
5. 「Setup was successful」と表示されればインストール完了です。

#### インストールの確認

コマンドプロンプトを開き、以下を入力して Python のバージョンが表示されることを確認してください。

```
python --version
```

`Python 3.12.x` のように表示されれば正常です。

---

### 3.2 アプリのインストール

#### exe ファイル版

1. car-diag の配布ページから `car-diag.exe` をダウンロードします。
2. ダウンロードした `car-diag.exe` を任意のフォルダに保存します（例: `C:\car-diag\`）。

インストール作業はこれだけです。別途 Python をインストールする必要はありません。

#### pip インストール版

コマンドプロンプトを開き、以下のコマンドを実行します。

```
pip install car-diag
```

依存ライブラリ（PyQt6、pyserial、pyqtgraph 等）が自動的にインストールされます。

---

### 3.3 初回起動

#### exe ファイル版の起動

`car-diag.exe` をダブルクリックします。

Windows セキュリティの警告（「Windows によって PC が保護されました」）が表示された場合は、「詳細情報」をクリックし、「実行」をクリックしてください。

#### pip インストール版の起動

コマンドプロンプトで以下を入力します。

```
car-diag
```

または

```
python -m src.main
```

#### 起動後の画面

起動すると、ウィンドウタイトルが「car-diag - ELM327 OBD-II Diagnostic Tool」のメインウィンドウが表示されます。ウィンドウ上部に「接続」「DTC」「ダッシュボード」「記録」の 4 つのタブがあります。ウィンドウ下部のステータスバーには「未接続」と表示されています。

## 4. 使い方

### 4.1 接続タブ

接続タブでは、ELM327 ドングルへの接続・切断と ECU のスキャンを行います。

#### 接続の手順

1. アプリを起動すると、「接続」タブが最初に表示されます。
2. 「COM ポート」グループの「ポート:」ドロップダウンから、[2.4 COM ポートの確認](#24-com-ポートの確認)で確認した COM ポートを選択します。
3. 一覧に目的のポートが表示されない場合は「**更新**」ボタンをクリックして一覧を再取得します。
4. 「**接続**」ボタンをクリックします。
5. ステータスバーに「接続中: COM○...」と表示され、初期化が始まります。
6. 接続が完了すると「接続完了: COM○ | PID数: ○○」と表示されます。

接続完了と同時に、車両が対応するパラメータ（PID）が自動的に検出されます。

接続に失敗した場合は、「接続失敗: ○○」と表示されます。「UNABLE TO CONNECT」などのメッセージが出た場合は、[5. トラブルシューティング](#5-トラブルシューティング)を参照してください。

#### ECU スキャンの手順

ECU スキャンを行うと、車両に搭載されている各 ECU（エンジン制御、ブレーキ制御など）を検出できます。DTC の取得や DID スキャンを行う前にスキャンを実行しておくことを推奨します。

1. ELM327 に接続した状態で「**ECU スキャン**」ボタンをクリックします。
2. スキャン中はステータスバーに「ECU スキャン中...」と表示されます。スキャンには数十秒かかる場合があります。
3. スキャン完了後、ステータスバーに「ECU スキャン完了: ○ 個検出」と表示されます。

ECU が 1 つも検出されなかった場合は、[5. トラブルシューティング](#5-トラブルシューティング)の「NO DATA が出る場合」を参照してください。

#### 切断の手順

「**切断**」ボタンをクリックするとELM327 との接続が切断されます。ステータスバーに「切断しました」と表示されます。

アプリを終了する場合は、先に「切断」をクリックすることを推奨します。

---

### 4.2 DTC タブ

DTC タブでは、車両の故障コード（DTC）の読取と消去を行います。

#### DTC とは

DTC（Diagnostic Trouble Code、診断トラブルコード）は、車両の ECU が検出した異常の種類を示すコードです。エンジン警告灯が点灯しているときは、1 つ以上の DTC が保存されていることがほとんどです。

DTC コードの先頭文字はシステムの種類を表します。

| 先頭文字 | 意味 |
|---------|------|
| P | パワートレイン（エンジン・トランスミッション） |
| C | シャシー（ブレーキ・サスペンション） |
| B | ボディ（エアバッグ・シート） |
| U | ネットワーク通信 |

#### DTC の読取手順

1. ELM327 に接続し、ECU スキャンを完了させておきます。
2. 「DTC」タブをクリックします。
3. 「**DTC 読取**」ボタンをクリックします。
4. 読取完了後、テーブルに結果が表示されます。テーブルの列は左から「ECU」「DTC コード」「説明」「ステータス」「プロトコル」です。

DTC が存在しない場合は「DTC なし」と表示されます。

#### DTC の説明の見方

「説明」列には、DTC コードに対応する英語の説明文が表示されます。

汎用の OBD-II コード（P0xxx、C0xxx、B0xxx、U0xxx）には説明文が表示されます。メーカー固有のコード（P1xxx など）は説明文が表示されず、「Unknown」と表示されます。

「ステータス」列には、DTC の状態を表す数値（例: `0x08`）が表示されます。

#### DTC の消去手順

> **重要な注意事項: DTC を消去する前に必ずお読みください**
>
> - DTC を消去すると、エンジン警告灯が消灯します。しかし、車両の異常が解消していない場合、走行すると再び DTC が記録され、警告灯が再点灯します。
> - DTC 消去後は、OBD-II の「レディネスモニター」がリセットされます。日本の車検（継続検査）では、レディネスモニターが完了状態でないと検査を通過できない場合があります。消去後は一定距離走行してモニターを完了状態にしてから車検に臨んでください。
> - 消去前に記録しておきたい DTC がある場合は、記録タブ（[4.4 記録タブ](#44-記録タブ)）でデータを保存してください。

1. ELM327 に接続し、ECU スキャンを完了させておきます。
2. **エンジンを停止した状態（イグニッションを ACC または ON にした状態）** にします。エンジンが動いている状態では消去できません。
3. 「DTC」タブで「**DTC 消去**」ボタンをクリックします。
4. 確認ダイアログが表示されるので、消去対象（全 ECU または個別 ECU）を選択して確認します。
5. 消去が完了すると「DTC 消去完了: ○ ECU 成功」と表示され、テーブルがクリアされます。

消去に失敗した場合（エラーメッセージが表示された場合）は、エンジンが停止しているか確認してください。

---

### 4.3 ダッシュボードタブ

ダッシュボードタブでは、車両のリアルタイムデータを表示します。

#### リアルタイムデータの見方

1. ELM327 に接続した状態で「ダッシュボード」タブをクリックします。
2. 「**モニタリング開始**」ボタンをクリックします。
3. 左側のテーブルに、車両が対応するパラメータの値がリアルタイムで更新されます。テーブルの列は「パラメータ」「値」「単位」「ECU」です。

表示されるパラメータは車両によって異なります。車両が対応していないパラメータは表示されません。

主なパラメータの例を以下に示します。

| パラメータ名 | 説明 | 単位 |
|------------|------|------|
| エンジン回転数 | エンジンの回転数 | rpm |
| 車速 | 車両の速度 | km/h |
| 冷却水温度 | エンジン冷却水の温度 | °C |
| スロットル開度 | アクセルペダルの踏み込み量 | % |
| バッテリー電圧 | バッテリーの電圧 | V |

4. モニタリングを停止するには「**モニタリング停止**」ボタンをクリックします。

#### グラフ表示の見方

画面右側にリアルタイムのグラフが表示されます。各パラメータの値が折れ線グラフで表示されます。グラフには直近約 5 分間（600 データ点）のデータが表示されます。

---

### 4.4 記録タブ

記録タブでは、リアルタイムデータをファイルに保存します。

#### データ記録の開始手順

データの記録はダッシュボードのモニタリングと連動しています。先にダッシュボードのモニタリングを開始しておくことを推奨します。

1. ELM327 に接続した状態で「記録」タブをクリックします。
2. 「**記録開始**」ボタンをクリックします。
3. ファイル保存ダイアログが表示されます。保存先のフォルダとファイル名を指定して「保存」をクリックします（拡張子は `.tsv` で保存されます）。
4. 「状態: 記録中」と表示され、記録が始まります。
5. 記録行数がリアルタイムで更新されます。

記録間隔は約 500ms（0.5 秒）を目標としています。

#### データ記録の停止手順

「**記録停止**」ボタンをクリックします。「状態: 停止」と表示され、ファイルが保存されます。

#### TSV ファイルの活用（Excel で開く）

記録したファイルは TSV 形式（タブ区切りテキスト）です。Excel で開くことができます。

**Excel で開く手順:**

1. Excel を起動します。
2. 「ファイル」→「開く」→「参照」をクリックします。
3. ファイルの種類を「テキスト ファイル (*.prn, *.txt, *.csv)」に変更します。
4. 保存した `.tsv` ファイルを選択して「開く」をクリックします。
5. テキストファイルウィザードが起動します。「区切り文字」で「タブ」にチェックを入れ、「完了」をクリックします。

1 行目がヘッダー行（タイムスタンプと各パラメータ名）、2 行目以降が記録データです。タイムスタンプは UTC 形式（ISO 8601）で記録されます。

---

### 4.5 DID スキャン

DID スキャンは、UDS（統一診断サービス）に対応した ECU が持つデータ識別子（DID）を探索する高度な機能です。

#### DID スキャンの前提条件

- ELM327 に接続し、ECU スキャンを完了させておく必要があります。
- UDS に対応した ECU が検出されている必要があります。

#### プリセットスキャン

プリセットスキャンは、よく使われる DID の範囲（F400-F4FF、F600-F6FF など）のみを対象とした短時間のスキャンです。

1. 接続タブで接続・ECU スキャンを完了させます。
2. プリセットスキャンを実行します（操作方法は画面の指示に従ってください）。
3. スキャン中は進捗ダイアログが表示されます。全体の進捗と各 ECU の進捗が 2 段のプログレスバーで表示されます。

#### 全スキャン

全スキャンは DID の全範囲（0x0000 〜 0xFFFF、65,536 件）を対象としたスキャンです。

> **所要時間の目安:** 全スキャンは 1 ECU あたり 30〜60 分かかります（リクエストあたり約 30〜50ms × 65,536 件）。ECU が複数ある場合は ECU 数に比例して時間が伸びます。実行前に推定所要時間が表示されるので、確認してから開始してください。

スキャンは途中で中断できます。中断ボタンをクリックすると、中断位置が自動的に保存されます。次回接続時に、続きから再開することができます。

スキャンで検出された DID の値はダッシュボードに表示されます。

## 5. トラブルシューティング

### 「UNABLE TO CONNECT」が表示される場合

ELM327 との通信が確立できないときに表示されます。以下の点を確認してください。

1. **ドングルが OBD-II ポートにしっかり差し込まれているか確認する。**
   いったん抜いて、カチッと音がするまで押し込み直してください。

2. **イグニッションが ACC または ON になっているか確認する。**
   ドングルに電源が供給されていないと通信できません。

3. **Bluetooth ペアリングが完了しているか確認する。**
   Windows の「Bluetooth とデバイス」設定でドングルが「接続済み」になっているか確認してください。ペアリングが切れている場合は再度ペアリングしてください（[2.3 Windows 11 での Bluetooth ペアリング](#23-windows-11-での-bluetooth-ペアリング)参照）。

4. **正しい COM ポートを選択しているか確認する。**
   デバイスマネージャーで COM ポート番号を確認し、アプリの「ポート:」ドロップダウンで同じ番号を選択してください（[2.4 COM ポートの確認](#24-com-ポートの確認)参照）。「更新」ボタンをクリックして一覧を再取得してから選択してください。

5. **ドングルがクローン品でないか確認する。**
   クローン品の ELM327 チップは一部のコマンドに対応していない場合があります（[2.1 ELM327 ドングルの購入ガイド](#21-elm327-ドングルの購入ガイド)参照）。

6. **再試行する。**
   ドングルを OBD-II ポートから抜き、数秒待ってから再度差し込んでください。その後、アプリの「接続」を再試行してください。

---

### 「NO DATA」が表示される場合

車両からデータが返ってこないときに表示されます。以下の点を確認してください。

1. **車両の OBD-II 対応を確認する。**
   日本国内販売の乗用車は 2001 年以降のモデルが対象ですが、一部の軽自動車は OBD-II に対応していない場合があります。

2. **イグニッションが ON になっているか確認する。**
   ACC 状態で通信できる場合もありますが、エンジンを始動した方が確実にデータが取得できる場合があります。

3. **ECU スキャンを先に実行する。**
   DTC の読取やダッシュボード表示を行う前に、接続タブの「ECU スキャン」を実行してください。

4. **プロトコルの切替が正常に行われているか確認する。**
   一部の車両では、CAN プロトコルと KWP2000 プロトコルが混在しています。ECU スキャン中にプロトコル切替が行われますが、時間がかかる場合があります。しばらく待ってから再試行してください。

---

### Bluetooth が繋がらない場合

1. **ドングルの電源を確認する。**
   ドングルが OBD-II ポートに正しく差し込まれ、イグニッションが ON になっているか確認してください。

2. **Windows の Bluetooth 設定を確認する。**
   「Bluetooth とデバイス」設定で Bluetooth がオンになっているか確認してください。

3. **ペアリングを削除して再ペアリングする。**
   「Bluetooth とデバイス」設定でドングルのデバイスを一度削除し、[2.3 Windows 11 での Bluetooth ペアリング](#23-windows-11-での-bluetooth-ペアリング)の手順で再ペアリングしてください。

4. **Bluetooth ドライバーを更新する。**
   デバイスマネージャーで Bluetooth アダプターのドライバーを最新版に更新してください。

5. **PC を再起動する。**
   Bluetooth の状態がおかしい場合は、PC を再起動することで解決する場合があります。

---

### DTC 消去ができない場合

エラーメッセージが表示されて消去できない場合は、以下を確認してください。

1. **エンジンが停止しているか確認する。**
   DTC 消去はエンジン停止中のみ実行できます（OBD-II 規格の仕様）。エンジンが動いている状態では消去コマンドが拒否されます。エンジンを切り、イグニッションを ACC または ON にした状態で再試行してください。

2. **ECU スキャンを先に実行する。**
   ECU スキャンが完了していない状態では DTC 消去ができません。

3. **エラーの内容を確認する。**
   「conditionsNotCorrect」などのエラーが表示された場合は、上記のエンジン停止の条件を満たしていない可能性があります。

## 6. FAQ

**Q. 車検前に DTC を消去してもいいですか？**

A. 消去自体は可能ですが、消去後すぐに車検を受けることはお勧めしません。DTC を消去すると OBD-II のレディネスモニターがリセットされます。車検（継続検査）ではレディネスモニターの完了状態が確認される場合があります。消去後は一定距離（目安: 数十キロ〜数百キロ）走行してモニターを完了状態に戻してから車検に臨んでください。

**Q. エンジン警告灯が消えません。**

A. DTC を消去してもエンジン警告灯がすぐに消えない場合があります。イグニッションを一度 OFF にして再度 ON にすると消灯する場合があります。それでも消えない場合は、車両の異常が解消されていない可能性があります。整備士に相談してください。

**Q. DTC の説明が「Unknown」と表示されます。**

A. car-diag が内蔵しているのは汎用 OBD-II コード（P0xxx、C0xxx、B0xxx、U0xxx）の説明文です。メーカー固有コード（P1xxx など）には説明文が表示されず「Unknown」となります。メーカー固有コードについては、車のディーラーや専門の整備士に確認してください。

**Q. ダッシュボードに表示されるパラメータが少ないです。**

A. 表示されるパラメータは車両が対応しているものだけです。車両によって対応パラメータの数は異なります。これは car-diag の問題ではありません。

**Q. 記録したデータのタイムスタンプの時刻がずれています。**

A. タイムスタンプは UTC（協定世界時）で記録されます。日本標準時（JST）は UTC+9 です。Excel などで分析する場合は 9 時間を加算してください。

**Q. DID スキャンを途中で止めましたが、データは保存されますか？**

A. はい。中断時点までの結果が自動的に保存されます。次回同じ車両（同じ VIN）に接続した場合、保存された結果を使用して続きからスキャンを再開できます。

**Q. 複数の車両で car-diag を使えますか？**

A. はい。car-diag は VIN（車両識別番号）ごとにスキャン結果をキャッシュとして保存します。異なる車両に接続すると、その車両のキャッシュが自動的に読み込まれます（VIN が取得できない場合は手動で識別名を設定します）。

**Q. キャッシュデータはどこに保存されますか？**

A. キャッシュは `C:\Users\{ユーザー名}\.car-diag\cache\` フォルダに JSON ファイルとして保存されます。90 日間アクセスがないキャッシュファイルは起動時に自動削除されます。

**Q. ELM327 が切断されたときのデータはどうなりますか？**

A. 接続が切断された場合、記録中のデータファイルは自動的に保存されて閉じられます。データの損失は最大 1 秒分以内に抑えられます。

## 7. 用語集

| 用語 | 説明 |
|------|------|
| OBD-II | On-Board Diagnostics II の略。車両の自己診断機能を定める国際規格。1996 年以降の米国販売車、および 2001 年以降の日本販売乗用車で義務化されている |
| ELM327 | ELM Electronics 社が開発した OBD-II インタプリタ IC。OBD-II の複雑なプロトコルを AT コマンドという簡単なコマンドで操作できるようにする |
| DTC | Diagnostic Trouble Code の略。診断トラブルコード。車両の ECU が検出した異常の種類を示す英数字のコード（例: P0300） |
| ECU | Electronic Control Unit の略。電子制御ユニット。エンジン制御、ブレーキ制御など車両の各システムを電子的に制御するコンピューター |
| PID | Parameter ID の略。パラメーター識別子。Legacy OBD-II で使用される、車両データの種類を示す番号（例: RPM、車速）。車両によって対応している PID の種類は異なる |
| DID | Data Identifier の略。データ識別子。UDS（ISO 14229）で使用される 2 バイトのパラメーター識別子 |
| MIL | Malfunction Indicator Lamp の略。エンジン警告灯（チェックエンジンランプ）。ECU が異常を検出すると点灯する |
| VIN | Vehicle Identification Number の略。車両識別番号。17 桁の英数字で構成される、車両固有の識別コード |
| COM ポート | Windows のシリアル通信ポート。Bluetooth デバイスは仮想 COM ポートとして認識される（例: COM3、COM7） |
| TSV | Tab-Separated Values の略。タブ区切りテキスト形式。Excel 等の表計算ソフトで開くことができる |
| Legacy OBD-II | ISO 15765-4 に基づく CAN バス上の標準診断プロトコル。日本の乗用車の多くで使われている |
| UDS | Unified Diagnostic Services の略。統一診断サービス（ISO 14229）。より高度な診断機能を提供するプロトコル |
| KWP2000 | Keyword Protocol 2000（ISO 14230）。K-Line という物理ケーブルで通信する、古い世代の診断プロトコル |
| レディネスモニター | OBD-II の排ガス診断機能が正常に動作したかを記録する項目。DTC 消去後にリセットされ、一定の走行条件を満たすと完了状態になる |
