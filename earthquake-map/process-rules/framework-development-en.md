# Framework Development Guide

> **Purpose of this document:** A guide for the development and maintenance of the gr-sw-maker framework itself. Intended for framework developers (those who revise the framework's rule documents and agent definitions).
> **Related documents:** [Porting Guide](porting-guide.md), [Agent List](agent-list.md), [Document Management Rules](full-auto-dev-document-rules.md)

---

## 1. Distinction Between Framework Development and App Development

gr-sw-maker has two usage modes:

| | Framework Development | App Development |
|---|---|---|
| **Purpose** | Revise the framework's own rules and agent definitions | Develop applications using gr-sw-maker |
| **Repository** | `gr-sw-maker` (the original on GitHub) | A project created with `npm init gr-sw-maker` |
| **Who** | Framework developers | App developers |
| **Tools used** | Does not use `create.js` / `setup.js` | `create.js` (initial setup) + `setup.js` (language selection) |

---

## 2. File and Folder Roles

This section defines how each file and folder is used in framework development versus app development.

### 2.1 Project Instruction Files

| File | Framework Development | App Development |
|---|---|---|
| `CLAUDE.md` | Edited directly. Instructions for framework development. `.gitignore` (private) | Generated by `setup.js` from `CLAUDE-ja/en.md`. Instructions for full-auto-dev. git tracked |
| `CLAUDE-ja.md` | Edited directly. App development template source (Japanese) | Copied to `CLAUDE.md` by `setup.js`. Retained as reference after copying |
| `CLAUDE-en.md` | Edited directly. App development template source (English) | Same as above |

**Important:** In framework development, `CLAUDE.md` and `CLAUDE-ja/en.md` are **separate files with different content**. `CLAUDE.md` contains work instructions for framework development, while `CLAUDE-ja/en.md` are template sources distributed to app developers.

### 2.2 Agent Definitions and Command Definitions

| File | Framework Development | App Development |
|---|---|---|
| `.claude/agents/*-ja.md` / `*-en.md` | Edited directly. Agent definition sources | Copied to unsuffixed versions by `setup.js` |
| `.claude/agents/*.md` (unsuffixed) | Not used. `.gitignore` | Generated by `setup.js`. git tracked |
| `.claude/commands/*-ja.md` / `*-en.md` | Used directly (council-review, etc.) | Copied to unsuffixed versions by `setup.js` |
| `.claude/commands/*.md` (unsuffixed) | Not used. `.gitignore` | Generated by `setup.js`. git tracked |

### 2.3 Process Rules

| File | Framework Development | App Development |
|---|---|---|
| `process-rules/*-ja.md` / `*-en.md` | Edited directly. Process rule sources | Copied to unsuffixed versions by `setup.js` |
| `process-rules/*.md` (unsuffixed) | Not used. `.gitignore` | Generated by `setup.js`. git tracked |

### 2.4 User Orders

| File | Framework Development | App Development |
|---|---|---|
| `user-order-ja.md` / `*-en.md` | Edited directly. Template sources | Copied to unsuffixed versions by `setup.js` |
| `user-order.md` (unsuffixed) | Not used. `.gitignore` | Generated by `setup.js`. For entering requirements. git tracked |

### 2.5 Distribution and Packaging

| File | Framework Development | App Development |
|---|---|---|
| `create-gr-sw-maker/` | Manages the npm package itself (`package.json` + `bin/create.js`) | Self-deletes after execution via `npm init gr-sw-maker` |
| `setup.js` | Not used. Managed as a distributable | `node setup.js <lang>` generates `.md` from `-ja/en.md` |

### 2.6 Others

| File | Framework Development | App Development |
|---|---|---|
| `.gitignore` | Excludes deploy outputs + `CLAUDE.md` + `prompt/` | `create.js` produces a version with deploy output exclusion lines removed |
| `README.md` | Edited directly. Framework description | `create.js` generates a template. App developers overwrite it |
| `README-ja.md` | Edited directly. Framework description (Japanese) | None (users create if needed) |
| `project-records/` | Framework review records, etc. | `create.js` deletes contents. App records accumulate here |
| `essays/` | Edited directly. Papers and research reports | Remains as-is (reference material) |
| `docs/` | Not used | Supplementary materials for app development |
| `prompt/` | Working notes, etc. `.gitignore` (private) | Does not exist |
| `settings.local.json` | Used. `.gitignore` | Used. `.gitignore` |

## 3. Roles of create.js and setup.js

### 3.1 create.js (`create-gr-sw-maker/bin/create.js`)

| Item | Details |
|---|---|
| **Trigger** | `npm init gr-sw-maker <project-name>` |
| **Usage context** | App development only |
| **Processing** | 1. Downloads the gr-sw-maker tarball from GitHub<br/>2. Extracts to the specified directory<br/>3. Deletes unnecessary files (`create-gr-sw-maker/`, contents of `project-records/`)<br/>4. Removes deploy output exclusion lines from `.gitignore`<br/>5. Generates `README.md` from a template |

### 3.2 setup.js

| Item | Details |
|---|---|
| **Trigger** | `node setup.js <lang>` (`ja` or `en`) |
| **Usage context** | App development only |
| **Processing** | Copies the selected language's `-ja.md` / `-en.md` to unsuffixed `.md` |
| **Target files** | `CLAUDE-*.md`, `.claude/agents/*-*.md`, `.claude/commands/*-*.md`, `user-order-*.md` |

## 4. The Dual Nature of .gitignore

`.gitignore` has different content in the framework repo versus an app project.

### 4.1 Framework Repo Version (Original)

```
# === Framework repo only (create.js removes everything below this line) ===

# Framework development private files (not distributed)
prompt/
CLAUDE.md

# setup.js deploy outputs
.claude/agents/*.md
!.claude/agents/*-ja.md
!.claude/agents/*-en.md
.claude/commands/*.md
!.claude/commands/*-ja.md
!.claude/commands/*-en.md
process-rules/*.md
!process-rules/*-ja.md
!process-rules/*-en.md
user-order.md
```

- Everything after the marker line (`# === Framework repo only ...`) is bulk-deleted by `create.js`
- `CLAUDE.md` (framework development instructions) and `prompt/` are private
- Deploy outputs (unsuffixed `.md`) are excluded
- `!*-*.md` restores `-ja.md` / `-en.md` sources to tracked status

### 4.2 App Project Version

Because `create.js` removes all lines after `# setup.js deploy outputs`, unsuffixed `.md` files become git tracked.

## 5. npm publish Procedure

Steps to publish a new version of the framework to npm:

1. Commit and push all framework changes
2. Run `npm publish` under `create-gr-sw-maker/`
3. Run E2E tests with `npm init gr-sw-maker <test-project>`
4. Verify that `node setup.js ja` and `node setup.js en` work correctly

**Note:** Only the `create-gr-sw-maker/` package is published to npm. The gr-sw-maker main body is not registered on npm (it is distributed via tarball download from GitHub).
